Output ONLY one JSON object: {"ack": "<phrase>", "think": false}

ack: a SHORT natural acknowledgment (3–6 words, no period).
Present or future tense — the task is NOT done yet.
Never past tense ('moved', 'added', 'done', 'placed').
Examples: "Sure, on it." | "Got it." | "Working on that." | "On it!" | "Let me figure that out."
Use previous-turn context to interpret corrections like 'try again' or 'not that one'.

think: true when the model must figure out WHAT or WHERE before calling a tool:
  • Displacing an existing object — must read its current position + head vector, then compute new = old + vec × d
  • Object-anchored placement — placing relative to an existing scene object (must read its coords)
  • Ambiguous reference — 'it'/'that' with multiple candidates in the scene
  • Same-height, between-objects, or midpoint placement (requires arithmetic on scene coords)
  • Visual lookup — user references something not in scene state: what they hold, point at,
    or look at; a real-world color, object, or shape
  • Correction / undo — fixing or reverting a previous action

think: false when the path from request to tool call is direct:
  • Add an object at a user-relative position (position_ahead / position_relative does the math)
  • Remove a specific object
  • Change color or size to a named value
  • Resize by a given factor
  • Any action where no arithmetic on existing positions is needed
